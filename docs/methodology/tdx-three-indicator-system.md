# 通达信三指标操作体系方法论初稿

研究对象：

1. 主图源码：综合趋势/K线染色/买卖图标/通道线。
2. 副图指标：MACD 变体 + 低位启动 + 持股/止盈动能线。
3. 水深指标：退出辅助因子，不作为主操作指标。

本文目标不是判断这套系统“必然有效”，而是先把公式语言翻译成可研究的交易方法论，再指出潜在问题和后续优化路径。

## 1. 总体判断

这套体系不是单指标交易系统，而是一个分层看盘体系：

- 主图：判断趋势状态、K 线结构、强势反包、短线卖点和通道位置。
- 副图：判断中短期动能、低位启动、持股状态和止盈观察。
- 水深：作为退出辅助，帮助判断是否进入“风险水位”，但不直接触发主操作。

更准确地说，它适合被整理成“半量化操作体系”：

```text
选股/观察：主图强势结构 + 副图低位启动
入场确认：主图趋势转强 + 副图动能转强
持仓管理：主图趋势颜色 + 副图持股线
止盈/退出：副图动能拐头 + 主图卖点 + 水深辅助
复盘验证：逐条信号拆开回测，不把整套视觉效果当结论
```

## 2. 三个指标的角色分工

### 2.1 主图：趋势与价格行为面板

主图包含多个功能模块：

| 模块 | 代表变量/信号 | 含义 |
|---|---|---|
| 行业展示 | `HYBLOCK` | 显示股票所属行业 |
| 核心趋势线 | `CXHZB` / `GZZ` | 加权均价趋势线及其 5 日平滑 |
| 趋势转强 | `CROSS(CXHZB,GZZ)` | 核心趋势线上穿，图标 7 |
| 趋势转弱 | `CROSS(GZZ,CXHZB)` | 核心趋势线下穿，图标 8 |
| 涨停识别 | `ZT` | 涨停 K 线高亮 |
| 强势反包 | `TJ` | 前阴放量后突破前高并站上 10 日均线 |
| 均线共振 | `FB` | 3 日、13 日、34 日等均线结构转强 |
| 区间强弱染色 | `VAR6` | 21 日高低区间内的相对强弱 |
| 动态通道 | `趋顶` / `趋底` | 平滑上下轨，类似压力/支撑 |
| 持股趋势线 | `牧马` / `操盘` | 价格站上 EMA10/EMA15 后显示 |
| 多空金叉 | `VAR11` | 多周期 EMA 平均线上穿其平滑线 |
| 短线风险 | `SWZB2` | 变形 RSI 高位回落，图标 28 |
| 短线卖点 | `卖` | 动量高位转弱，图标 6 |

主图的核心作用是：把价格行为和趋势状态可视化。

在方法论里，主图不宜直接作为“一键买卖”系统，而应该拆成三类信号：

#### A. 入场候选

- `CROSS(CXHZB,GZZ)`：趋势转强。
- `TJ`：强势反包/突破前高。
- `FB`：短均线共振转强。
- `VAR11`：多周期 EMA 动能金叉。

这些信号适合用于“加入观察”或“触发入场检查”，不建议单独构成买点。

#### B. 持仓确认

- `CXHZB >= GZZ`
- `牧马` 出现
- `操盘` 出现
- K 线处于红色/强势染色状态

这些条件说明趋势仍在。它们更适合做“继续持有”的背景条件。

#### C. 退出/风险提示

- `CROSS(GZZ,CXHZB)`：核心趋势转弱。
- `SWZB2`：高位动能转弱。
- `卖`：强动能冲高后拐头。
- 价格接近或突破 `趋顶` 后出现动能转弱。

主图的退出信号偏短线，最好与副图止盈和水深一起确认。

### 2.2 副图：动能与持仓管理

副图比主图更接近可量化交易逻辑。它主要由三个部分组成：

#### A. 筹码辅助

```tdx
获利%:WINNER(CLOSE)*100;
平均成本:COST(50);
```

- `获利%`：当前价格下获利盘比例。
- `平均成本`：50% 筹码成本。

这两个变量适合做背景解释，不适合直接做买卖触发。原因是筹码函数依赖通达信内部算法，跨平台复现难度较高。

#### B. MACD 变体

```tdx
CLZ = EMA(C,12) - EMA(C,26)
XHX = EMA(CLZ,9)
ZHZ = 2 * (CLZ - XHX)
```

这本质上就是 MACD 柱体：

- 红柱扩张：正动能增强。
- 紫柱收缩：正动能减弱。
- 黄柱修复：负动能减弱。
- 绿柱恶化：负动能增强。

可转化为事件：

| 事件 | 公式语言 | 含义 |
|---|---|---|
| 正动能扩张 | `ZHZ>=0 AND ZHZ>=REF(ZHZ,1)` | 强势延续 |
| 正动能收缩 | `ZHZ>=0 AND ZHZ<REF(ZHZ,1)` | 止盈观察 |
| 负动能修复 | `ZHZ<0 AND ZHZ>=REF(ZHZ,1)` | 弱转强观察 |
| 负动能恶化 | `ZHZ<0 AND ZHZ<REF(ZHZ,1)` | 回避 |

#### C. 低位启动与止盈

```tdx
天马 = CROSS(AZ3,AZ4) AND AZ3<20
持股 = 观变>=REF(观变,1)
止盈 = 观变<=REF(观变,1)
```

含义：

- `天马`：34 周期低位区间内的慢速 KDJ 金叉，偏低位启动信号。
- `持股`：MACD DIF 平滑中枢继续上升。
- `止盈`：MACD DIF 平滑中枢开始下降。

副图是这套体系里最适合做量化测试的部分。

建议把副图信号定义为：

```text
entry_low_start = 天马
momentum_hold = 观变 >= REF(观变,1)
take_profit_watch = 观变 <= REF(观变,1)
macd_positive_shrink = ZHZ >= 0 AND ZHZ < REF(ZHZ,1)
macd_negative_break = CROSS(0, ZHZ)
```

### 2.3 水深：退出辅助因子

水深公式：

```tdx
VAR1:=0.618/3;
VAR2:=2*HIGH-LOW/3;
VAR3:=IF(CLOSE>55,618,61.8)/3;
水深: LOG(VAR1)*DMA(EMA(CLOSE,55),SUM(VOL,55))*EXP(VAR2)/POW(VAR3,6)/2;
```

这个指标不适合作为主操作指标，原因是：

- `EXP(VAR2)` 对价格绝对值极端敏感。
- `DMA(EMA(CLOSE,55),SUM(VOL,55))` 的权重写法非常反常。
- `CLOSE>55` 是价格绝对阈值，不同股价区间不可直接比较。
- 输出值可能被放大到极端，解释性弱。

但如果它只是“辅助退出指标”，可以这样定位：

- 不直接决定卖出。
- 只在主图/副图已出现止盈或风险信号时，作为二次确认。
- 不比较不同股票之间的水深绝对值，只比较单只股票自身的历史分位、拐点和极值。

建议把水深转成以下事件，而不是直接用数值：

```text
water_extreme = 水深处于自身 N 日极端分位
water_turn = 水深出现方向拐点
water_divergence = 股价创新高但水深未同步改善/恶化
water_confirm_exit = 副图止盈 + 主图卖点 + 水深极端或拐头
```

## 3. 操作体系方法论

### 3.1 观察阶段

股票进入观察池，应满足至少一种强势或低位启动特征：

- 主图 `CROSS(CXHZB,GZZ)`。
- 主图 `TJ` 强势反包。
- 副图 `天马`。
- 副图 ZHZ 由绿转黄，负动能开始修复。

观察阶段不等于买入，只说明“值得看”。

### 3.2 入场阶段

建议使用“双确认”：

```text
入场候选 = 主图趋势转强 OR 主图强势反包 OR 副图天马
入场确认 = 副图观变上升 AND ZHZ 不恶化
过滤条件 = 不追连续大涨后高位主图卖点，不在水深异常风险区强行买入
```

更严格版本：

```text
买入 = (CROSS(CXHZB,GZZ) OR TJ OR VAR11)
      AND 观变 >= REF(观变,1)
      AND ZHZ >= REF(ZHZ,1)
```

低位版本：

```text
低位买入观察 = 天马
确认买入 = 天马后 N 日内，主图趋势转强，且观变持续上升
```

### 3.3 持仓阶段

持仓的核心不是“出现买点后死拿”，而是持续确认趋势没有破坏。

持仓条件：

- 副图 `持股` 仍显示，即 `观变>=REF(观变,1)`。
- 主图 `CXHZB>=GZZ`。
- 主图 `牧马` 或 `操盘` 存在。
- ZHZ 正柱扩张或负柱修复。

只要这些条件大部分成立，短线波动不应轻易卖出。

### 3.4 止盈观察阶段

出现以下任一情况，进入止盈观察，不一定立刻卖：

- 副图 `止盈` 出现，即 `观变<=REF(观变,1)`。
- ZHZ 红柱转紫柱，正动能收缩。
- 主图 `SWZB2` 出现。
- 股价接近 `趋顶` 后出现副图动能收缩。
- 水深进入自身极端区或出现拐头。

止盈观察的动作可以是：

- 减仓。
- 提高止损线。
- 次日不再加仓。
- 等待主图卖点确认。

### 3.5 退出阶段

建议把退出分三档：

#### 轻度退出：减仓或停止加仓

```text
副图止盈 OR ZHZ正柱收缩 OR 主图SWZB2
```

#### 中度退出：卖出一部分

```text
副图止盈
AND 主图卖点/趋势转弱之一
```

#### 强退出：清仓或大幅降低仓位

```text
CROSS(GZZ,CXHZB)
OR ZHZ跌破0
OR 主图卖点 + 副图止盈 + 水深风险确认
```

水深在这里只做“确认器”，不单独触发强退出。

## 4. 潜在问题

### 4.1 主图使用 `XMA`，存在未来函数风险

主图的 `趋顶`、`趋底` 使用：

```tdx
XMA(XMA(H,25),25)
XMA(XMA(L,25),25)
```

`XMA` 在很多通达信体系里有未来函数风险，可能导致历史图上看起来很准，但实盘信号漂移。

处理方式：

- 回测时单独标记含 `XMA` 的信号。
- 第一版严谨回测不把 `趋顶/趋底` 作为买卖条件，只作为人工视觉参考。
- 后续可用 `EMA`、`MA`、`HHV/LLV` 通道替代测试。

### 4.2 主图信号太多，容易过拟合

主图同时包含趋势转强、涨停、反包、均线共振、通道、短线卖点。若全部叠加，很容易变成“看图解释”，不利于回测。

处理方式：

- 拆成独立信号逐个测试。
- 每次只测试一个入场信号 + 一个退出信号。
- 不要一开始测试完整视觉系统。

### 4.3 水深数值尺度不稳定

水深使用 `EXP(VAR2)` 和价格绝对阈值 `CLOSE>55`，对股价区间极敏感。

处理方式：

- 不用水深绝对值跨股票比较。
- 只用单股自身滚动分位。
- 对水深做 `log_abs`、z-score 或 winsorize 后再研究。
- 先确认公式是否有抄写错误，例如 `VAR2` 是否原意为 `(2*HIGH-LOW)/3`。

### 4.4 筹码函数难以跨平台复现

`WINNER`、`COST` 依赖通达信内部筹码算法，离开通达信不容易复现。

处理方式：

- 在通达信/TdxQuant 内计算，不自己重写。
- 回测时保存函数输出，不只保存最终信号。
- 如果后续迁移到其他数据源，应把筹码变量视为不可迁移特征。

### 4.5 信号确认存在滞后

副图 `止盈` 基于 `观变<=REF(观变,1)`，本质是动能拐头，可能晚于价格高点。

处理方式：

- 测试“首次止盈观察”与“二次确认退出”的差异。
- 对趋势股允许滞后，对短线股需要更快退出。
- 分别统计少赚、错杀、保利润能力。

## 5. 优化方向

### 5.1 先做信号字典

把所有信号标准化为事件：

| 信号名 | 来源 | 类型 | 用途 |
|---|---|---|---|
| `trend_cross_up` | 主图 | 入场候选 | `CROSS(CXHZB,GZZ)` |
| `trend_cross_down` | 主图 | 退出候选 | `CROSS(GZZ,CXHZB)` |
| `strong_reversal` | 主图 | 入场候选 | `TJ` |
| `ema_multi_cross` | 主图 | 入场候选 | `VAR11` |
| `top_momentum_sell` | 主图 | 退出候选 | `卖` |
| `rsi_rollover_sell` | 主图 | 止盈观察 | `SWZB2` |
| `low_start` | 副图 | 入场候选 | `天马` |
| `momentum_hold` | 副图 | 持仓 | `观变>=REF(观变,1)` |
| `momentum_take_profit` | 副图 | 止盈观察 | `观变<=REF(观变,1)` |
| `water_exit_confirm` | 水深 | 退出辅助 | 水深极端/拐头/背离 |

### 5.2 建立三种策略版本

#### V1：副图纯动能版

最干净，便于验证。

```text
买入：天马 OR ZHZ由负转正
持有：观变上升
卖出：观变下降 或 ZHZ跌破0
```

#### V2：主图趋势确认版

加入主图趋势过滤。

```text
买入：副图买入候选 AND 主图趋势转强
卖出：副图止盈 AND 主图趋势转弱
```

#### V3：水深退出增强版

只测试水深是否改善退出质量。

```text
买入：沿用 V2
卖出：V2卖出 OR (副图止盈 AND 水深风险确认)
```

对比 V2 和 V3，才能判断水深是否真的有价值。

### 5.3 回测指标

每个版本至少看：

- 交易次数。
- 胜率。
- 平均收益/中位收益。
- 最大回撤。
- 平均持有天数。
- 单笔最大亏损。
- 错杀率：卖出后 N 日继续上涨超过阈值。
- 保护率：卖出后 N 日下跌超过阈值。
- 和买入持有、随机退出、普通 MACD 退出对比。

### 5.4 水深优化

水深不要直接使用原始值。建议派生：

```text
water_rank_120 = 水深在过去120日分位
water_slope_3 = 水深3日变化
water_zscore_120 = 水深120日标准化
water_extreme_turn = 分位极端后拐头
```

重点验证：

- 它是否能提前于副图止盈。
- 它是否能减少大回撤。
- 它是否只是价格绝对值的变形。
- 它在不同价位股票上是否稳定。

## 6. 初步结论

这套体系有可研究价值，但要拆开看：

- 副图最适合作为第一版量化核心，因为逻辑接近 MACD/KDJ，解释性强。
- 主图适合作为趋势和价格行为确认，但含 `XMA` 的通道部分要谨慎。
- 水深只能作为退出辅助，必须用自身分位/拐点测试，不能直接看绝对数值。

下一步不应急着接实盘，而是先做“信号转写 + 历史回测”：

1. 把三段公式拆成信号字典。
2. 先测试副图 V1。
3. 再加主图确认 V2。
4. 最后测试水深是否改善退出 V3。
5. 只有 V3 明确优于 V2，水深才保留在体系里。
